## プロダクト概要

Kali Linux の **インストール～環境構築～実利用** で発生する問題・疑問を、
ユーザーの問いかけに対する「原因」と「対策」として整理し、**静的 HTML 化** して
Q&A 形式の知識ベースとしてリポジトリに蓄積するプロジェクト。サイトタイトルは
**『KaliLinuxことはじめ』** で固定する。

### 厳守すべき制約

- **想定読者は Linux コマンドライン初心者**。コマンド名だけ挙げて済ませず、
  プロンプト付き入力例・期待される対話／出力・前提となるターミナル操作への
  導線まで記事に含める
- サイトタイトルは **『KaliLinuxことはじめ』** で固定する
- 出力先は **リポジトリ直下の `html/` フォルダ**。記事は静的 HTML として保存する
- 公開先（GitHub Pages／XServer／自身の WordPress）は **現時点で未定**。
  どのホスティングへも持っていけるよう、依存の少ない **純粋な静的 HTML/CSS/JS** で
  書き、サーバ側処理を必要としない構成にする
- ページ間リンクは **相対パス** で書く（公開先のパス構造が変わっても破綻させない）
- 認証情報・実環境設定（API キー、`~/.ssh/*`、`.env` 等）はリポジトリにコミットしない
- **先回りで大量の記事を勝手に作らない**。ユーザーが個別の問いかけをするたび、
  その回答を記事化する受動型のワークフローを基本とする

### 完了条件

1. 各記事ページが **ブラウザでローカル単独表示** できる（オフラインでも主要情報が読める）
2. 各記事に **「質問」「原因」「対策」** の 3 要素が揃って記載されている
3. サイトトップ（`html/index.html` 等）から個別記事ページへ辿れる導線がある
4. 機能・論理単位（記事追加・サイト構造変更など）の区切りで
   `git commit` & `git push origin` を実施する

## 使用言語

本リポジトリ配下での生成テキストはすべて日本語で書く。対象は以下を含む。

- ユーザーへのチャット返信、進捗報告、ターン末の要約
- `AskUserQuestion` の質問文・選択肢のラベルと説明
- コード内のコメント（`//`, `#`, `/* */`, docstring 等）
- コミットメッセージ、PR タイトル、PR 本文
- README、CLAUDE.md、設計メモ、その他のドキュメント類
- エラーメッセージやログ出力のうち、エンドユーザー向けに表示される文字列

例外として **英語のまま** とするのは次のみ（言語仕様・エコシステム慣習のため）。

- プログラミング言語のキーワード、標準ライブラリ／フレームワーク／パッケージの API 名
- 識別子（変数名・関数名・クラス名・ファイル名・ディレクトリ名など）
- 設定ファイルのキー名、環境変数名
