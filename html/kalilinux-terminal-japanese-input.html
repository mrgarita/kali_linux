KaliLinuxことはじめ

Kali Linux のターミナルや vi / nano で日本語入力する（Qt 用 fcitx5 フロントエンドを追加する）

質問

日本語入力と日本語表示の設定記事の手順を最後まで実施して、Mousepad などの GUI エディタでは Ctrl+Space で「あ」⇔「A」が切り替わり日本語入力できる状態になった。しかし Kali Xfce のデフォルトターミナル qterminal を開いてシェルプロンプトの状態で Ctrl+Space を押しても、「あ」インジケータが出ない・日本語入力モードに切り替わらない。vi や nano を起動して入力しようとしても英数字だけが入力される。先に Mousepad 側で「あ」モードにしておいてから vi を起動しても、vi 上は英数入力になる。環境変数 GTK_IM_MODULE / QT_IM_MODULE / XMODIFIERS はターミナルで確認したところ正しく設定されている（それぞれ fcitx / fcitx / @im=fcitx）。なぜ動かないのか、どこを直せばよいか。

原因（理由）

fcitx5 がアプリケーションに日本語入力を提供する仕組みは 3 層構造 になっていて、3 層すべてが揃わないと動かない。既存記事「日本語入力と日本語表示」の手順では そのうちの 2 層しか入れていない ため、Qt アプリ（qterminal）が呼べる入力モジュールが存在せず、ターミナル全体で日本語入力ができない状態になっている。

  • 第 1 層：fcitx5 本体（fcitx5 パッケージ）。入力メソッドフレームワーク。既存記事手順 3 で導入済み。
  • 第 2 層：変換エンジン（fcitx5-mozc パッケージ）。ローマ字を「日本語」に変換する Mozc エンジン。既存記事手順 3 で導入済み。
  • 第 3 層：各 GUI ツールキット用の「フロントエンド」（入力モジュール）。アプリ側から fcitx5 を呼び出す具体的なライブラリで、ツールキットごとに別パッケージ：
    • GTK 用 → fcitx5-frontend-gtk3 / fcitx5-frontend-gtk4
    • Qt 用 → fcitx5-frontend-qt5 / fcitx5-frontend-qt6
    • 古典的な XIM 用 → fcitx5-frontend-xim（古い X アプリ向け）
    これが既存記事の手順では入っていなかった。

環境変数 QT_IM_MODULE=fcitx は「Qt アプリは fcitx5 を使え」と宣言する役割で、これだけでは入力できない。アプリが「fcitx5 を呼べ」と聞いたあとに 実体である IM モジュール（fcitx5-frontend-qt5 など）が存在しないと、ライブラリが見つからずスキップされてしまう。Kali Xfce のデフォルトターミナル qterminal は Qt アプリなので、Qt 用フロントエンドが入っていない状態では fcitx5 を呼び出せない。シェルプロンプトでも、その上で動く vi / nano でも、入力経路は同じ「qterminal → 子プロセス」なので、ターミナル自体が fcitx5 を受け取れない限り中で動くものすべて日本語入力できない。

なお紛らわしい点として、fcitx5-config-qt（既存記事手順 3 で入れた）は Qt 製の設定 GUI ツール（fcitx5-configtool コマンド）であって、Qt アプリ全般に入力を提供するフロントエンドとは別物。名前が似ているが役割が違う。

対策（手順）

1. 現状を確認する

まず fcitx5 本体が動作していて、環境変数も正しいことを再確認する（ここがズレていると別問題なので最初に切り分け）。

kali@kali:~$ pgrep -l fcitx5
1323 fcitx5
kali@kali:~$ echo $GTK_IM_MODULE
fcitx
kali@kali:~$ echo $QT_IM_MODULE
fcitx
kali@kali:~$ echo $XMODIFIERS
@im=fcitx

3 つの環境変数がそれぞれ fcitx / fcitx / @im=fcitx になっていれば、fcitx5 を使う宣言は OK。違う値や空の場合は 既存記事「日本語入力と日本語表示」の手順 3 に戻り、im-config -n fcitx5 を実行してから再ログインする。

使用中のターミナル種別も確認しておく：

kali@kali:~$ ps -p $PPID -o comm=
qterminal

qterminal（Qt 系）か、xfce4-terminal（GTK 系）か、gnome-terminal-（GTK 系）かで、必要なフロントエンドが変わる。Kali Xfce のデフォルトは qterminal。

2. 既存のフロントエンド導入状況と apt の取得可能パッケージを確認する

すでに入っている fcitx5 関連パッケージを一覧する：

kali@kali:~$ apt list --installed 2>/dev/null | grep -i fcitx5
fcitx5-config-qt/kali-rolling,now 5.1.13-2 amd64 [インストール済み]
fcitx5-data/kali-rolling,now 5.1.19-1 all [インストール済み、自動]
fcitx5-modules/kali-rolling,now 5.1.19-1 amd64 [インストール済み、自動]
fcitx5-mozc/kali-rolling,now 2.29.5160.102+dfsg-1.6 amd64 [インストール済み]
fcitx5/kali-rolling,now 5.1.19-1 amd64 [インストール済み]
libfcitx5-qt-data/kali-rolling,now 5.1.13-1 all [インストール済み、自動]
libfcitx5-qt6-1/kali-rolling,now 5.1.13-1+b1 amd64 [インストール済み、自動]
libfcitx5config6/kali-rolling,now 5.1.19-1 amd64 [インストール済み、自動]
libfcitx5core7/kali-rolling,now 5.1.19-1 amd64 [インストール済み、自動]
libfcitx5utils2/kali-rolling,now 5.1.19-1 amd64 [インストール済み、自動]

注目すべき点：

  • fcitx5-frontend-qt5 / fcitx5-frontend-qt6 / fcitx5-frontend-gtk3 / fcitx5-frontend-gtk4 など fcitx5-frontend-* が一切入っていない（これが原因）
  • libfcitx5-qt6-1 は入っているが、これは Qt6 用の 共有ライブラリであって IM モジュールではない。フロントエンドパッケージとは別物
  • fcitx5-config-qt は設定 GUI ツール（fcitx5-configtool コマンド）であり、Qt アプリ全般に入力を提供するフロントエンドではない

続いて apt 上で取得可能な fcitx5 関連のフロントエンドを確認する：

kali@kali:~$ apt-cache search fcitx5 | grep -E 'qt|gtk'
fcitx5-config-qt - configuration tool for Fcitx5 (Qt version)
fcitx5-frontend-gtk2 - GTK2 IM Module for fcitx5
fcitx5-frontend-gtk3 - GTK3 IM Module for fcitx5
fcitx5-frontend-gtk4 - GTK4 IM Module for fcitx5
fcitx5-frontend-qt5 - Qt5 IM module for fcitx5
fcitx5-frontend-qt6 - Qt6 IM module for fcitx5
libfcitx5-qt-data - Qt library and IM module for fcitx5 (data files)
libfcitx5-qt-dev - Qt library and IM module for fcitx5 (development files)
libfcitx5-qt1 - Qt library and IM module for fcitx5
libfcitx5-qt6-1 - Qt6 library and IM module for fcitx5
libfcitx5-qt6-dev - Qt6 library and IM module for fcitx5 (development files)

Kali Rolling の現行リポジトリでは、fcitx5-frontend-qt5 / fcitx5-frontend-qt6 / fcitx5-frontend-gtk3 / fcitx5-frontend-gtk4 が個別パッケージとして提供されている（Ubuntu 等にある fcitx5-frontend-all のような「全部入りメタパッケージ」は 存在しないので、手動で複数指定する）。

3. Qt 用と GTK 用のフロントエンドをまとめてインストールする

Qt 用と GTK 用の最新世代（Qt5/Qt6・GTK3/GTK4）を 1 行でまとめて導入する。qterminal 単体目的なら fcitx5-frontend-qt6 だけでも動くが、将来別のアプリ（GTK 系のターミナルやエディタなど）も使う可能性を考えると、4 つまとめて入れておくのが手戻りが少ない。

kali@kali:~$ sudo apt install -y fcitx5-frontend-qt5 fcitx5-frontend-qt6 fcitx5-frontend-gtk3 fcitx5-frontend-gtk4
  • fcitx5-frontend-qt6 = 本記事の問題を直接解決する本命パッケージ。qterminal など Qt6 アプリ用
  • fcitx5-frontend-qt5 = Qt5 製アプリ（VLC など）用の保険
  • fcitx5-frontend-gtk3 = xfce4-terminal、Mousepad、Firefox など GTK3 アプリ用
  • fcitx5-frontend-gtk4 = 新しめの GTK4 アプリ用

※ fcitx5-frontend-gtk2 は GTK2 という古い世代向けで、現代の Kali では入れる必要がない。

インストール後、もう一度 apt list --installed で確認しておくと安心：

kali@kali:~$ apt list --installed 2>/dev/null | grep fcitx5-frontend
fcitx5-frontend-gtk3/kali-rolling,now ... [インストール済み]
fcitx5-frontend-gtk4/kali-rolling,now ... [インストール済み]
fcitx5-frontend-qt5/kali-rolling,now ... [インストール済み]
fcitx5-frontend-qt6/kali-rolling,now ... [インストール済み]

4 つとも [インストール済み] と表示されていれば導入完了。

4. fcitx5 を再起動して新フロントエンドを認識させる

フロントエンドを入れただけでは、すでに動いているアプリ（特に fcitx5 本体と、現在開いている qterminal）はまだ新しいモジュールを知らない。再起動して読み込ませる。最も確実なのは OS ごとログアウトして再ログイン：

  • 画面右上のメニュー（ユーザー名／電源アイコン） →「ログアウト」→ ダイアログでさらに「ログアウト」
  • lightdm のログイン画面に戻ったら、再度ログインする

ログアウトせずに済ませたい場合は、fcitx5 プロセスだけ再起動する：

kali@kali:~$ fcitx5 -r &

-r は再起動オプション、末尾の & はターミナルに制御を戻すためのバックグラウンド実行指定。実行直後にプロセス番号らしき表示が 1 行出るが、そのまま Enter でプロンプトに戻る。ただし、すでに開いている qterminal ウィンドウは その時点では新フロントエンドを取り込まない（プロセス起動時に IM モジュールを読み込むため）。fcitx5 -r したあと、新しい qterminal ウィンドウを開き直してそちらで試す。

5. 動作確認

新しく開いた qterminal のシェルプロンプトで、まずインジケータが出るかを確認する：

kali@kali:~$

この状態で Ctrl+Space を押す。カーソル付近に小さなポップアップで「あ」が出れば、ターミナル経由で fcitx5 が呼ばれている。試しに nihongo と打ち Space で「日本語」に変換される。Enter 確定で、シェルプロンプトの位置に 日本語 という文字列が入力される（コマンドとしては不正なので Ctrl+C で取り消してよい）。

続いて vi で日本語入力を試す：

kali@kali:~$ vi /tmp/japanese-test.txt

vi が開いたら：

  1. i を押して挿入モード（画面下に -- INSERT -- 表示）に入る
  2. Ctrl+Space でインジケータを「あ」に切り替え
  3. nihongo と打ち Space で「日本語」に変換、Enter で確定
  4. Esc でノーマルモードに戻り、:wq と打ち Enter で保存終了

書き込んだ内容を確認：

kali@kali:~$ cat /tmp/japanese-test.txt
日本語

nano でも同様に確認できる：

kali@kali:~$ nano /tmp/japanese-test2.txt
  • nano は起動直後がそのまま入力状態（モード切替なし）
  • Ctrl+Space で「あ」モードへ → nihongo → Space → Enter
  • Ctrl+O → Enter で保存、Ctrl+X で終了

すべて日本語が入力できれば対策完了。

注意点

  • ターミナルで日本語が 表示 されるためには、ロケールが ja_JP.UTF-8 で、日本語フォント（fonts-noto-cjk）が入っていること。これは 日本語入力と日本語表示の記事の手順 1 で済んでいるはず。表示が □（豆腐）になる場合はそちらに戻る
  • fcitx5 -r & で再起動した直後、すでに開いている qterminal ウィンドウでは新フロントエンドが効かない。プロセス起動時に IM モジュールを読み込む仕組みなので、必ず 新しい qterminal ウィンドウを開き直してから動作確認する。確実に全アプリへ反映させるならログアウト → 再ログインが安全
  • fcitx5-config-qt（設定ツール、fcitx5-configtool コマンド）と fcitx5-frontend-qt5 / fcitx5-frontend-qt6（入力モジュール）は 別パッケージ。前者は Qt で作られた設定 GUI、後者は Qt アプリ全般に入力を提供する実体。名前が似ているので混同しないこと
  • Kali Rolling の現行リポジトリには fcitx5-frontend-all のような全部入りメタパッケージは 存在しない（他ディストリの解説記事で見かけるが、Kali では使えない）。手順 3 のように個別パッケージを並べて指定する
  • vi / nano が 書き込んだファイルが文字化けする場合、ファイル側のエンコーディングが UTF-8 でない可能性がある。file /tmp/japanese-test.txt で確認し、UTF-8 でなければ iconv -f <元エンコーディング> -t UTF-8 <ファイル> で変換する
  • Vim 8 以降は既定で UTF-8 を扱えるので ~/.vimrc の追加設定は通常不要。古い設定が残っていて文字化けする場合のみ ~/.vimrc に set encoding=utf-8 と set fileencoding=utf-8 を書く
  • tmux や screen 経由で動かしている vi / nano では、tmux 側の入力ハンドリングが介在するため、本記事の対策だけでは解決しないことがある。tmux の set -g default-terminal "xterm-256color" や setw -g utf8 on（古い tmux）等の設定を別途確認する
  • ssh でリモートホストに接続している場合、リモート側に fcitx5 は 必要ない。ローカルの fcitx5 がローカルの qterminal 上で動作し、変換後の日本語文字列が ssh 経由でリモートシェルへ送信される構造。リモート側に日本語ロケールが入っていれば表示・入力ともに問題ない
  • Wayland セッションでは入力周りの挙動が異なる。Kali Xfce のデフォルトは X11 なので通常気にしなくてよいが、確認したい場合は echo $XDG_SESSION_TYPE を実行（x11 なら本記事の手順通り）
  • qterminal ではなく xfce4-terminal（GTK 系）や gnome-terminal を主に使う環境でも、手順 3 で fcitx5-frontend-gtk3 / fcitx5-frontend-gtk4 を入れていれば同じ要領で日本語入力できる
