KaliLinuxことはじめ

VirtualBox が必要な理由と Windows 11 へのインストール手順

問い

Kali Linux を使うのに VirtualBox のインストールが前提となっている記事が多い。 なぜ VirtualBox が必要なのか、そして Windows ではどうインストールすればよいのか。

原因（理由）

  • Kali Linux は攻撃検証用のツールを多数同梱しており、普段使いの OS（Windows）に直接インストールすると、ホスト環境を汚しやすい／必要なときだけ立ち上げる運用がしづらい／誤操作のリスクがある
  • 公式ドキュメントも 仮想化環境内での実行 を標準的な利用形態として案内している
  • VirtualBox を使えば Windows をホスト OS のまま、Kali Linux をゲスト OS として隔離して動かす 構成が取れる
  • 数ある仮想化ソフトの中で VirtualBox を選ぶ理由：無料／オープンソース／Kali 公式が動作確認している／設定が比較的シンプル

対策（Windows 11 でのインストール手順）

  1. ブラウザで VirtualBox 公式サイト を開き、Downloads から Windows hosts 版の最新インストーラ（.exe）をダウンロードする
  2. ダウンロードした .exe を右クリック →「管理者として実行」
  3. インストーラの画面指示に従い Next で進める（インストール先・機能選択は既定値で問題ない）
  4. インストール途中で ネットワーク機能の導入による一時的なネットワーク切断 の警告が出る → Yes で続行
  5. インストール完了後、スタートメニューに Oracle VM VirtualBox が追加される
  6. （任意）公式サイトから VirtualBox Extension Pack もダウンロードし、VirtualBox 起動後の ファイル → ツール → 拡張機能パッケージ マネージャー から追加する（USB 2.0/3.0、リモートデスクトップなどの追加機能が利用できる）

注意点

  • BIOS／UEFI で Intel VT-x または AMD-V が有効 になっている必要がある（無効だと VM が起動しない）
  • Windows の Hyper-V が有効 だと VirtualBox の動作が遅くなる場合がある（Hyper-V 無効化の手順は別記事として用意する予定）
  • Windows 10 でも基本手順は同じ。インストーラの UI 文言が微妙に異なる程度
