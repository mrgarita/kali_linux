KaliLinuxことはじめ

Kali Linux で初めてのポートスキャン — Nmap で scanme.nmap.org を調べる

質問

Kali Linux の環境が整ったので、定番ツール Nmap を最初に触ってみたい。ただ、いきなり 実在の他人のサイトに使うのは怖い。安全に練習できる相手はどこにあるのか。

また、教わったとおり nmap scanme.nmap.org を実行したら、次のような警告が出て いつまでも終わらない。これは故障なのか、どう読み解けばよいのか。

kali@kali:~$ nmap scanme.nmap.org
Starting Nmap 7.99 ( https://nmap.org ) at 2026-05-31 09:51 +0900
Warning: 45.33.32.156 giving up on port because retransmission cap hit (10).

原因（理由）

  • Nmap のポートスキャンは「相手の入口（ポート）を外から調べる」行為。許可なく他人のサーバへ向けると、法律（日本では不正アクセス禁止法など）やレンタルサーバの規約に触れるおそれがある。だから練習には「スキャンしてよい」と明示的に許可されたホストを使う
  • その公式の練習用ホストが scanme.nmap.org。Nmap 開発元が「学習目的のスキャンを許可する」と公開しているサーバで、最初の一歩に最適
  • 「終わらない」のは故障ではない。Nmap のデフォルトは主要 1000 ポートを調べるが、その大半が filtered（ファイアウォールが探索パケットを黙って捨て、応答が返らない状態）だと、Nmap は返事を待って何度も送り直し（retransmission）を繰り返す。retransmission cap hit (10) は「あるポートに 10 回送り直しても返事がないので諦めた」という情報メッセージで、これが多発すると全体が極端に遅くなる
  • 結果は PORT / STATE / SERVICE の表で読む。STATE が open＝接続できる入口、closed＝閉じている、filtered＝遮られて判別できない、を表す

対策（手順）

1. 練習相手は scanme.nmap.org を使う

実在の他人のサイトではなく、Nmap 公式が許可している scanme.nmap.org に対して練習する。自分が所有・運用するサーバ以外には向けない、というのが大原則。

2. Nmap が入っているか確認する

kali@kali:~$ nmap --version
Nmap version 7.99 ( https://nmap.org )

バージョンが表示されれば準備完了（Kali には標準で入っている）。$ はプロンプトなので入力しない。

3. まずは基本のスキャン（遅いことがある）

kali@kali:~$ nmap scanme.nmap.org

主要 1000 ポートを調べる。scanme は世界中の人が練習で使う混雑したホストで、回線状況によっては前述の retransmission cap hit 警告が出て、なかなか終わらないことがある。これは正常な挙動。

4. 実行中に「進んでいるか」を確認する／中断する

スキャン中に Enter など適当なキーを 1 回押すと、進捗の一行が表示される：

Stats: 0:01:30 elapsed; 0 hosts completed (1 up), 1 undergoing SYN Stealth Scan
SYN Stealth Scan Timing: About 42.13% done; ETC: 09:55 (0:02:05 remaining)

remaining（残り時間）を見て、長すぎると感じたら Ctrl+C で安全に中断できる。

5. 終わらないときは -F で対象ポートを絞る

-F（fast）を付けると、主要 100 ポートだけを手早く調べる。-v は途中経過を表示するオプション。

kali@kali:~$ nmap -v -F scanme.nmap.org
Starting Nmap 7.99 ( https://nmap.org ) at 2026-05-31 10:15 +0900
Initiating Ping Scan at 10:15
Completed Ping Scan at 10:15, 0.01s elapsed (1 total hosts)
Initiating SYN Stealth Scan at 10:15
Scanning scanme.nmap.org (45.33.32.156) [100 ports]
Discovered open port 22/tcp on 45.33.32.156
Discovered open port 80/tcp on 45.33.32.156
Completed SYN Stealth Scan at 10:15, 2.49s elapsed (100 total ports)
Nmap scan report for scanme.nmap.org (45.33.32.156)
Host is up (0.027s latency).
Not shown: 98 filtered tcp ports (no-response)
PORT   STATE SERVICE
22/tcp open  ssh
80/tcp open  http

Nmap done: 1 IP address (1 host up) scanned in 2.80 seconds
           Raw packets sent: 203 (8.904KB) | Rcvd: 5 (208B)

1000 ポートではなく 100 ポートに絞ったので、わずか数秒で完了した。

6. 出力の読み方

  • Host is up (0.027s latency) … 相手は生きていて、往復約 0.027 秒（速い）
  • Not shown: 98 filtered tcp ports (no-response) … ここが「遅さ」の正体。100 ポート中 98 ポートが filtered（応答なし）。デフォルトの 1000 ポートではこの“応答なし待ち”が大量に発生するため遅くなる
  • 表の open 行が成果。22/tcp ssh と 80/tcp http の 2 つが「外から接続できる入口」
  • 最終行 Raw packets sent: 203 | Rcvd: 5 … 203 個送って返ってきたのは 5 個だけ。ほとんど無視（filtered）されたことを裏付けている
STATE意味
openポートが開いていて接続を受け付けている（＝入口）
closed到達はするが、そのポートは閉じている
filteredファイアウォール等に遮られ、開閉を判別できない

7. 開いていた入口を -sV で深掘りする（バージョン特定）

開いていた 2 ポートに絞り、-sV（version）で「そこで動いているソフトと正確なバージョン」まで調べる。-sV は精度のため管理者権限が要るので sudo を付ける（初回はパスワードを聞かれる。入力中は画面に何も表示されないが、そのまま打って Enter）。

kali@kali:~$ sudo nmap -sV -p 22,80 scanme.nmap.org
[sudo] kali のパスワード:
Starting Nmap 7.99 ( https://nmap.org ) at 2026-05-31 10:27 +0900
Nmap scan report for scanme.nmap.org (45.33.32.156)
Host is up (0.031s latency).

PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 6.6.1p1 Ubuntu 2ubuntu2.13 (Ubuntu Linux; protocol 2.0)
80/tcp open  http    Apache httpd 2.4.7 ((Ubuntu))
Service Info: OS: Linux; CPE: cpe:/o:linux:linux_kernel

Nmap done: 1 IP address (1 host up) scanned in 8.67 seconds
  • -p 22,80 … 調べるポートを 2 つに限定
  • 右端の VERSION 列が -sV の成果。OpenSSH 6.6.1p1 と Apache httpd 2.4.7 という具体的なソフトと版が判明した
  • Service Info: OS: Linux … バナーから中身は Linux と推測。CPE: はソフトを世界共通の書式で表した ID で、脆弱性データベースと照合する際のキーになる

8. バージョンが分かると何ができるのか

ソフト名とバージョンが特定できると、「その版に既知の弱点（脆弱性）があるか」を調べる次の工程へ進める。具体的には、脆弱性データベース（NVD/CVE）でその版を検索したり、Kali 内蔵の searchsploit で既知の手法を探したりする。つまりステップ 1 の Nmap は、後続の脆弱性スキャンの“地図”を作る工程だと分かる。

注意点

  • 許可された対象にだけ実行する。練習は scanme.nmap.org、または自分が所有・運用するサーバに限る。他人のサーバへのスキャンは違法・規約違反になり得る
  • scanme.nmap.org は混雑しているため遅い・警告が出るのが普通。故障ではない
  • -sV は相手のポートに少しデータを送って会話する。単に眺めるだけの完全に受動的な操作ではない点に留意する
  • 自分のサイト（例：レンタルサーバ上の WordPress）に向ける場合は、(1) 契約サーバの規約でポートスキャンが禁止されていないか確認する、(2) Cloudflare などの CDN を挟んでいると、実サーバではなく CDN を調べてしまう、という 2 点に注意する
