Kali Linux のインストール手順（VirtualBox 前提）と必要容量

質問

VirtualBox はすでにインストール済み。ここから Kali Linux を VirtualBox 上に乗せたい。 具体的な手順と、ディスク容量はどれくらい必要なのか。また Kali Linux はかなり容量を食うらしいが、その理由も知っておきたい。

原因（理由）

Kali 公式は 事前構築済みの VirtualBox イメージ を配布しており、これを使うのが最短経路。 ISO から新規 VM を作って手動インストールする方法もあるが、初回は事前構築イメージで十分間に合う。

容量について：実機で計測したところ、VirtualBox 本体と Kali Linux を合わせて 約 36 GB 必要になった。内訳の目安：

  • VirtualBox 本体（実行ファイル群）：数百 MB
  • Kali 配布 .7z アーカイブ（ダウンロード時）：約 3〜4 GB
  • 展開後の仮想ディスク（.vdi）＋ 起動・更新で増えた分：30 GB 前後

Kali Linux がここまで大容量になる理由：

  • 600 種類以上のセキュリティ／ペネトレーションテスト用ツール を最初からプリインストールしている（Nmap、Metasploit、Burp Suite、Wireshark、John the Ripper、Hydra、Aircrack-ng、Hashcat など）
  • パスワード解析用のワードリスト（rockyou.txt など）や、Metasploit のモジュール／ペイロードのデータベースを同梱している
  • フォレンジック・リバースエンジニアリング系ツール（Autopsy、Volatility、Ghidra、radare2 など）も標準搭載
  • Python／Ruby／Perl／Go など、各種ツールが依存する 複数の言語ランタイムと豊富なライブラリ を同時に抱える
  • デフォルトの Xfce デスクトップ環境 と、日本語フォント等の言語パック

不要なツールを後から削れば縮められるが、学習段階では「全部入り」のまま使うほうが手っ取り早い。

対策（手順：事前構築済みイメージを使う最短経路）

  1. ブラウザで Kali 公式の Virtual Machines ダウンロードページ を開く
  2. VirtualBox 64-bit の .7z ファイル（ファイル名例：kali-linux-YYYY.X-virtualbox-amd64.7z）をダウンロードする
  3. 7-Zip などの解凍ソフトで .7z を展開する（展開先のフォルダに .vbox 定義ファイルと仮想ディスク .vdi が入っている）
  4. 展開先フォルダの .vbox ファイルをダブルクリック → 自動的に VirtualBox が起動し、VM 一覧に Kali Linux が登録される
  5. VirtualBox の VM 一覧で Kali Linux を選択し、「起動」 をクリック
  6. 初回ログインは ユーザー名 kali ／ パスワード kali（ログイン後、端末で passwd を実行してパスワードを変更しておく）
  7. 起動確認後、最新化のため端末で次を実行：
    sudo apt update && sudo apt full-upgrade -y（管理者パスワードは初期値の kali）

注意点

  • ダウンロード前後で SHA256 ハッシュ を公式ページの値と照合し、.7z が改ざんされていないかを確認しておくと安心
  • 展開先・配置先のドライブには 40 GB 以上の空き を見ておくと安全（アップデート・ログ・スナップショットで容量は増えていくため）
  • 初回起動が遅い／黒画面で止まる場合は、BIOS／UEFI の 仮想化支援機能（Intel VT-x／AMD-V）が有効か、Windows の Hyper-V が無効か、を再確認する（前記事 VirtualBox が必要な理由と Windows 11 へのインストール手順 末尾の注意点を参照）
  • 本記事の手順は 事前構築済みイメージ による最短経路。ISO からのカスタムインストールが必要になった場合は別記事で扱う
