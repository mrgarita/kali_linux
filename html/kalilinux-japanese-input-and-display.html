Kali Linux で日本語入力と日本語表示を有効にする

質問

Kali Linux をインストールした直後は、メニューや端末出力が英語のままで日本語が表示されず、文字を打っても日本語入力ができない。さらに、fcitx5 を入れて日本語入力を準備しても、JIS キーボードを使っているのにキートップの刻印と実際に出る記号がずれる（@ を押すと " が出る等）、日本語と英語の切替キーが分からない、といった二次的な問題が出てくる。これらをまとめてどう設定すればよいか。

原因（理由）

  • Kali のデフォルトロケールは en_US.UTF-8 で、日本語ロケール（ja_JP.UTF-8）や日本語フォント（Noto CJK 等）が初期状態では入っていない
  • 日本語入力エンジン（IME）も初期状態では入っていない。fcitx5 本体と日本語変換エンジン（mozc）の両方を別々にインストールし、さらに 入力メソッドとして fcitx5 を使う ことを OS に教える操作が必要
  • さらに落とし穴として、fcitx5-mozc パッケージを apt で入れただけでは Mozc が fcitx5 の「現在の入力メソッド」一覧に自動追加されない ことがある（環境・バージョンによる）。fcitx5-configtool から手動で Mozc を追加しないと、fcitx5 自体は動いていても呼び出せる入力メソッドが英語キーボードだけになり、日本語入力ができない
  • インストール時にキーボードレイアウトを English (US) のまま進めると、JIS（日本語）配列のキーボードを物理的に使っていても、OS は US 配列として記号位置を割り当てる。これがキー刻印と入力結果がずれる原因
  • fcitx5 の既定切替キーは Ctrl+Space のため、JIS キーボード左上の 半角/全角 キーは押しても反応しない。半角/全角 で切り替えたい場合は fcitx5 / Mozc 側で 明示的にキーを割り当てる 必要がある

対策（手順）

1. 日本語フォントとロケールを入れる

まず現在のロケール設定を確認する：

kali@kali:~$ locale
LANG=en_US.UTF-8
LANGUAGE=
LC_CTYPE="en_US.UTF-8"
...
LC_ALL=

日本語フォントと日本語関連パッケージをまとめて導入する：

kali@kali:~$ sudo apt update
kali@kali:~$ sudo apt install -y task-japanese fonts-noto-cjk fonts-noto-cjk-extra
  • task-japanese = Debian が用意している「日本語環境に必要なパッケージ一式」の取りまとめ
  • fonts-noto-cjk / fonts-noto-cjk-extra = 日本語・中国語・韓国語の文字を表示するためのフォント。これがないと □（豆腐）になる

次に、日本語ロケールを生成して既定にする：

kali@kali:~$ sudo dpkg-reconfigure locales

青い画面（テキスト UI のダイアログ）が出るので、次の通り操作する：

  • 長い言語リストから ja_JP.UTF-8 UTF-8 を見つけ、Space で チェックを入れる（行頭が [*] になる）。Tab で OK にカーソルを移し Enter
  • 続いて「デフォルトのロケールはどれにするか」を聞かれるので ja_JP.UTF-8 を選んで Enter

反映するには再ログインまたは再起動が必要：

kali@kali:~$ sudo reboot

再起動後、再度 locale や date を実行して日本語表示になっているか確認する：

kali@kali:~$ locale
LANG=ja_JP.UTF-8
...
kali@kali:~$ date
2026年  5月 27日 水曜日 15:30:45 JST

2. キーボードレイアウトを JIS（日本語配列）に合わせる

JIS キーボードを使っているのに @ や _ がキー刻印と違う位置から出る場合は、OS 側のキーボードレイアウトが US 配列のままになっている。Xfce では GUI とコマンドの 2 通り で変更できる。

方法 A：GUI で変更する（推奨）

  • 画面左上「アプリケーション」メニュー →「設定」→「キーボード」を開く（Xfce の Settings Manager 経由でも可）
  • 「レイアウト」タブを選ぶ
  • 「システムのデフォルトを使用する」のチェックを外す
  • レイアウト一覧の下にある「追加」ボタンを押し、一覧から Japanese →「Japanese (OADG 109A)」を選んで「OK」
  • 既存の English (US) 行を選んで「削除」ボタンで取り除く（Japanese が唯一のレイアウトになる）
  • 設定画面下の入力欄にキーを試し打ちして、刻印どおりに記号が出ることを確認

方法 B：コマンドで恒久的に変更する

GUI が見つからない／使えないときは、設定ファイルを書き換える対話形式のコマンドを使う：

kali@kali:~$ sudo dpkg-reconfigure keyboard-configuration

順番に次のように選ぶ（青い画面で ↑↓ で選択、Enter で確定）：

  • キーボードモデル：Generic 105-key PC (intl.)（特殊キーが付いた機種ならその機種名）
  • レイアウト国：Japanese
  • キーボードレイアウト：Japanese - Japanese (OADG 109A)
  • AltGr キー、Compose キー：いずれも既定のままで Enter

設定はその場で /etc/default/keyboard に書かれ、再起動後も維持される。GUI のレイアウトとは別に 仮想コンソール（Ctrl+Alt+F2 等で出る黒い画面）の配列も同じ設定で揃う ので、両方一致させたいときはこちらが確実。

方法 C：いま開いているセッションだけ一時的に切り替えたい

kali@kali:~$ setxkbmap -layout jp

再起動すると元に戻る。動作確認用と割り切って使う。

動作確認：以下のキーを試し打ちして、刻印どおりの記号が出るかチェックする。

  • @（P の右隣） → @ が出る
  • Shift+2 → " が出る（US 配列だと @ が出てしまう）
  • Shift+- → =
  • Shift+ろ（右下） → _

3. 日本語入力（fcitx5 + mozc）を入れる

本体・変換エンジン・設定 GUI をまとめて導入する：

kali@kali:~$ sudo apt install -y fcitx5 fcitx5-mozc fcitx5-config-qt
  • fcitx5 = 入力メソッドフレームワーク本体
  • fcitx5-mozc = Mozc（Google 日本語入力のオープンソース版）を fcitx5 から使うためのモジュール
  • fcitx5-config-qt = fcitx5 の設定ツール（fcitx5-configtool コマンドで起動）

続いて、OS の入力メソッドとして fcitx5 を使うことを宣言する：

kali@kali:~$ im-config -n fcitx5

このコマンドは 成功しても画面には何も出ない（無音で ~/.xinputrc に設定が書き込まれる）。エラーが出ていなければそのまま次へ進む。

反映には再ログインか再起動が必要：

kali@kali:~$ sudo reboot

再起動後、fcitx5 が起動していることを確認する：

kali@kali:~$ pgrep -l fcitx5
1323 fcitx5

このように番号と fcitx5 が表示されれば、自動起動は成功。何も表示されない場合は本記事末尾の「注意点」を参照。

続けて、fcitx5 に Mozc を「使う入力メソッド」として登録する。これをやらないと fcitx5 は動いていても英語キーボードしか呼び出せず、日本語入力ができない（実機で必ず詰まる箇所）。端末から設定ツールを起動：

kali@kali:~$ fcitx5-configtool

または「アプリケーション」メニュー →「設定」→「Fcitx 5 設定」からも開ける。開いたウィンドウは次のような左右パネル構成になっている：

  • 左パネル：現在のグループのツリー。Default グループの下に Keyboard - English (US) が並んでいる
  • 右パネル：「有効な入力メソッド」リスト。Mozc を含む追加候補が並ぶ
  • 中央：左右のパネルの間に「←」「→」の矢印ボタンがある

Mozc を Default グループに追加する手順：

  1. 右パネルの「有効な入力メソッド」リストから Mozc をシングルクリックで選択する
  2. 左右パネルの間にある中央の「←」ボタン（左向き矢印）を押す
  3. 左パネルの Default 配下に Keyboard - English (US) と Mozc の両方が並んでいることを目視で確認
  4. 右下の「適用」ボタンを押して設定を反映
  5. 続いて右下の「閉じる」ボタンでウィンドウを閉じる（右下のボタンは「適用」「OK」「閉じる」の 3 つが並ぶが、「OK」を押しても ウィンドウが閉じない ことがあるので明示的に「閉じる」を押す）

動作確認：テキストエディタ（メニュー →「アクセサリ」→ Mousepad など）を開き、次の挙動になるか確かめる：

  • Ctrl+Space を押すと、入力中のカーソルのすぐ下に小さなポップアップで「あ」（日本語入力）／「A」（直接入力）が表示される。パネル右上ではなく カーソル付近にポップアップで出る点 に注意（パネルに見当たらなくても慌てない）
  • 「あ」状態のまま nihongo と打ち Space を押すと「日本語」に変換される

ここまで動けば、IME のセットアップは完了。

4. 半角/全角 キーで切り替えられることを確認する（任意）

前提：手順 3 まで完了し、テキストエディタで Ctrl+Space により「あ」⇔「A」が切り替わる状態になっていること。

Kali / Debian の fcitx5 は 既定で 半角/全角 キーも切替キーとして登録済み なので、JIS キーボードならば追加設定なしで使えるはず。まず確認だけする：

  1. テキストエディタ（Mousepad など）を開き、入力欄にカーソルを置く
  2. JIS キーボード左上の 半角/全角 キーを単独で押す
  3. カーソル下のポップアップが「あ」⇔「A」と切り替わればそれで完了

切り替わらない場合：既定の「全角半角」エントリは登録されているのにキーコードの内部表現がずれて反応しないことがある（実機で発生した既知のパターン）。次の手順で登録し直す：

  1. 端末から設定ツールを起動： 
    kali@kali:~$ fcitx5-configtool
  2. ウィンドウ上部のタブから「グローバルオプション」を選ぶ
  3. 「ホットキー」セクションの「入力メソッドの切り替え:」項目を見る
  4. 既定で Control+スペース、全角半角、ハングル の 3 つが縦に並んでいる。各行の右側にある「-」（削除）「+」（追加）ボタンを使う
  5. 既存の 全角半角 行の右の「-」ボタンを押して、いったん削除する
  6. 同じ項目の「+」ボタンを押すと、「キーを押してください」状態のダイアログが表示される
  7. JIS キーボードの 半角/全角 キーを 1 回押す。値として再び 全角半角 が記録される
  8. 右下の「適用」→「閉じる」でウィンドウを閉じる

もう一度テキストエディタで 半角/全角 を単独で押すと、「あ」⇔「A」が切り替わるようになる（実機で確認済み）。

注意点

  • VirtualBox 上で Kali を動かしている場合、ホスト OS（Windows 等）の IME と Kali の fcitx5 が同時に反応して二重入力になることがある。日本語入力するときは ホスト側の IME を OFF にしてから ゲストウィンドウへフォーカスを移す
  • ロケールを日本語にした後でも、sudo 経由で実行したコマンドのエラーメッセージは英語のまま出ることがある。これは sudo が安全のため環境変数を引き継がない既定挙動。日本語で見たいときは sudo -E コマンド のように -E を付ける
  • dpkg-reconfigure keyboard-configuration は GUI セッションと仮想コンソール（Ctrl+Alt+F2 等の黒い画面）の両方の配列を変える。GUI だけ揃えたい場合は Xfce の「キーボード」設定（方法 A）だけにしておく
  • 半角/全角 キーは VirtualBox の「ホストキー」と衝突する設定にはなっていない（既定は右 Ctrl）が、自分でホストキーを変えている場合は競合に注意
  • fcitx5 と旧来の ibus は同時に有効化できない。過去に ibus を入れたことがある環境では sudo apt purge ibus* でいったん削除してから fcitx5 を入れ直すと衝突を避けられる
  • fcitx5-mozc を apt で入れただけでは、Mozc が fcitx5 の入力メソッド一覧に並ばないことがある（自動追加されるかは環境・バージョン依存）。Ctrl+Space を押しても切替インジケータが English のままだったら、まず fcitx5-configtool を開いて左パネルの Default グループに Mozc が並んでいるか確認する
  • fcitx5-configtool の右下のボタンは「適用」「OK」「閉じる」の 3 つが並ぶ。版によっては「OK」を押してもウィンドウが閉じないことがある。閉じないときは「閉じる」ボタンを押す
  • IME 切替時のインジケータ（「あ」「A」）は、パネル右上ではなく 入力中のカーソル付近にポップアップで出る ことが多い。パネルに見当たらなくても慌てず、テキスト入力欄にフォーカスを合わせて Ctrl+Space を押し、カーソル下にポップアップが出るかで判断する
  • 再起動後にパネルにも fcitx5 の表示が無く、入力切替も効かないように見えるときは、端末で pgrep -l fcitx5 を実行する。プロセスが表示されれば fcitx5 自体は動いている。それでも切替できない場合は fcitx5-configtool で入力メソッド一覧を確認する（最頻パターンは Mozc 未登録）
  • 「グローバルオプション」タブの「入力メソッドの切り替え」「入力メソッドを有効にする」「入力メソッドをオフにする」などの項目に既定で出ている ハングル / ハングル Hanja / ハングル Romaja は 韓国語キーボード用のキー名。JIS キーボードには物理的に該当キーが無いので押すこともなく、残しておいても日本語入力に影響しない。リストをすっきりさせたい場合は「-」ボタンで削除してもよい
