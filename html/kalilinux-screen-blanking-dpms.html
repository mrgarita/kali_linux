KaliLinuxことはじめ

Kali Linux の画面がすぐに真っ黒になる（DPMS と電源管理を止める）

質問

Kali Linux にログインしたあと、キーボードもマウスも操作しないでいると数分で画面が真っ黒になる。動画の視聴中や、コマンドの実行結果をぼんやり眺めているときにも勝手に消えてしまう。ターミナルから xset -dpms を打つと、その場では消灯しなくなる（このセッションが続く限りは効く）。そこで「ログインのたびに自動でこのコマンドを実行させたい」と思って、~/.bashrc・~/.profile・~/.zprofile・~/.zshrc・/etc/rc.local など思いつく初期化ファイルに片っ端から書き足してみた。しかし再ログインや再起動の後に確認するとまた消えるようになっている。どこに書けば毎回自動で効くようにできるのか。

原因（理由）

「書き場所がぜんぶ違っていた」のと「xset の設定とは別経路で画面を消しに来るプロセスがいる」の 2 つが同時に起きている。とくに後者は 「ディスプレイ電源管理（DPMS）を OFF にしたのに、まだ 5 分くらいで黒くなる」 の正体になる。順に整理する。

  • ~/.bashrc / ~/.zshrc は対話シェル起動時にしか読まれない。GUI でログインしてデスクトップ（Xfce など）が立ち上がる経路では実行されない。だからここに xset -dpms を書いても、「ターミナルを開いた瞬間だけ」効く形になり、GUI セッション全体には反映されない。
  • ~/.profile / ~/.zprofile は「ログインシェル」用。Kali の既定の lightdm（GUI ログイン画面）から Xfce セッションを開始する経路では、シェルではなく X セッションスクリプトが起動するため、これらのファイルは通常読まれない（テキストログインからの startx など、限られた経路でしか実行されない）。
  • /etc/rc.local は X が起動するより前に systemd から実行される。xset は X サーバに対して話しかけるコマンドなので、まだ X が無い段階で実行されると unable to open display エラーで失敗してそのまま終わる。DISPLAY 環境変数も認証情報（~/.Xauthority）もない段階だからどうやっても効かない。
  • そもそも画面を黒くしてくる仕組みが 4 系統あり、別々に止めないと止まらない。現行 Kali Xfce では次の 4 つが独立して動いている：
    • X 組み込みの DPMS（Display Power Management Signaling）。モニタへ省電力信号を送る。xset -dpms で無効化する。
    • X 組み込みのスクリーンセーバ。xset s off（タイマー停止）と xset s noblank（ブランク描画停止）で止める。DPMS と独立。
    • Xfce の電源管理デーモン xfce4-power-manager。DPMS の有効化／タイムアウトを独自に管理し、ログイン後に xset の設定を上書きしてくる。「電源管理」設定アプリで OFF にする。
    • ★ Xfce のスクリーンセーバ xfce4-screensaver（最大の主犯）。xfce4-power-manager とは 完全に別のパッケージ／別のプロセス。デフォルトで有効になっており、アイドル 5 分 でテーマ「ブランク画面」を表示するため画面が真っ黒になる。xset s off も xfce4-power-manager 側の設定も ここには効かない。これを止めない限り「DPMS を OFF にしてもまだ黒くなる」状態が続く。
  • light-locker や xscreensaver が動いている環境もある。xfce4-screensaver を入れる前の Kali や、ユーザーが意図的に古典的なスクリーンセーバを入れた環境では、これらが xfce4-screensaver の代わりに（あるいは併存して）画面をブラックアウト＆ロックする。

つまり「正しい場所に書く」だけでなく、「画面を消しに来るプロセスを 2 系統まとめて 黙らせる」ことが必要。以下の対策では、まず GUI で xfce4-power-manager を切り（DPMS の上書き元を止め）、続いて xfce4-screensaver を切り（5 分でブランク画面を出すプロセスを止め）、最後に ~/.xsessionrc で xset 系の保険を入れる、という 3 段構えにする。

対策（手順）

1. 現状を確認する

まずは xset q で「いま DPMS とスクリーンセーバがどう設定されているか」を見る。ここで読めるようになっておくと、対策後に「ちゃんと無効化できているか」を自分で確認できる。

kali@kali:~$ xset q

主要な出力部分（抜粋）：

Screen Saver:
  prefer blanking:  yes    allow exposures:  yes
  timeout:  600    cycle:  600
...
DPMS (Display Power Management Signaling):
  Standby: 600    Suspend: 600    Off: 600
  DPMS is Enabled
  Monitor is On

読み方：

  • Screen Saver: timeout: 600 = 600 秒（10 分）何も操作しないと X 組み込みスクリーンセーバが発動する
  • prefer blanking: yes = 発動時に画面を真っ黒に塗りつぶす（これが「黒くなる」の正体の一つ）
  • DPMS is Enabled = モニタ電源管理が有効。Standby/Suspend/Off の各秒数で段階的にモニタへ省電力信号を送る

対策後にもう一度 xset q を実行して、DPMS is Disabled／timeout: 0／prefer blanking: no になっていれば「X 側」は無効化できたという確認になる。

注意：xset q には xfce4-screensaver の状態は 映らない。xset q 上は無効化されているように見えても画面が黒くなる場合は、原因は xfce4-screensaver 側にあるので、手順 3 を必ず実施する。

2. Xfce の電源管理を GUI で無効化する

Kali のデフォルトデスクトップ Xfce では、xfce4-power-manager が DPMS を独自に管理している。これが xset の設定を上書きしてくるので、まずここを止める。

「アプリケーション」メニュー →「設定」→「電源管理」を開く。コマンドからも起動できる：

kali@kali:~$ xfce4-power-manager-settings

「電源管理」ウィンドウは上部に 「一般」「システム」「ディスプレイ」「デバイス」 の 4 タブが並ぶ。「ディスプレイ」タブを開く。

「ディスプレイ」タブの中はさらに 「バッテリ駆動」「AC 接続」のサブタブ に分かれており、デスクトップ機・ノート機どちらでも両方の設定欄が存在する。両方のサブタブで 次の操作を行う：

  • 欄一番上の「ディスプレイ電源管理」右端のトグルスイッチを押して OFF（灰色側）にする
  • OFF にすると下の「スリープするまでの時間」「電源を切断するまでの時間」のスライダは自動的に無効化（灰色表示）される。スライダ自体の値はそのままで構わない
  • サブタブを「AC 接続」に切り替え、同じくトグルを OFF にする（バッテリ駆動／AC 接続のどちらか一方しか OFF にしないと、もう一方の電源状態のときに発動する）

右下の「閉じる」を押すと設定はその場で保存され、再ログイン後も保持される。

確認のため、もう一度 xset q を実行する：

kali@kali:~$ xset q

出力中の DPMS セクションが DPMS is Disabled に変わっていれば、xfce4-power-manager 側はもう DPMS を有効化しに来ない。

ただし、ここまでで止まるのは「DPMS によるモニタ電源オフ」だけ。多くの環境では xfce4-screensaver が別途動いていて、こちらが アイドル 5 分でブランク画面（真っ黒）を表示 してくる。「DPMS を切ったのにまだ 5 分くらいで黒くなる」場合は手順 3 を続けて実施する。

3. Xfce のスクリーンセーバを GUI で無効化する

こちらが 「DPMS を OFF にしてもまだ黒くなる」の本丸。xfce4-screensaver は xfce4-power-manager とは別パッケージ・別プロセスで、設定 GUI も別ウィンドウ。

「電源管理」ウィンドウの 下部にある「スクリーンセーバーの管理(S)」 ボタンを押す（先ほどの「電源管理」ウィンドウをまだ開いているならそのまま、閉じてしまった場合は手順 2 で開き直す）。コマンドから直接開くこともできる：

kali@kali:~$ xfce4-screensaver-preferences

別ウィンドウ「スクリーンセーバーの設定」が開く。タブは 「スクリーンセーバー」「ロック画面」 の 2 つ。

「スクリーンセーバー」タブで次のどちらかを OFF にする：

  • 一番上の「スクリーンセーバーを有効にする」右端のトグルを押して OFF にする（推奨。スクリーンセーバ機能そのものを止める）
  • または、中段の「コンピューターがアイドル状態のときにスクリーンセーバーをアクティブにする」トグルを OFF にする（アイドルでの自動発動だけを止め、機能自体は残す）

テーマ一覧で「ブランク画面」が選ばれていることが、5 分後に「真っ黒」になっていた直接の原因（ブランク画面 = 画面を黒一色で塗りつぶすテーマ）。トグルを OFF にすれば、テーマ選択に関係なく発動しなくなる。

「ロック画面」タブには「ロック画面を有効にする」「スクリーンセーバーで画面をロックする」「システムスリープで画面をロックする」などのトグルが並ぶが、「スクリーンセーバー」タブの親トグルを OFF にした時点でスクリーンセーバ自体が動かなくなるため、これらは触らなくても自動的にロック発動も止まる（実機検証済み）。画面ロック機能そのものを完全に無効化したい場合だけ、一番上の「ロック画面を有効にする」を OFF にする。学習用 VM では好みで、共用 PC や業務用途では残す判断もあり（本記事末尾の「注意点」参照）。

右下の「閉じる」で確定。設定はその場で反映され、ログイン後も保持される。

確認：ターミナルや Mousepad を開いた状態で 5 分以上放置し、画面が黒くならなければ成功。あわせてターミナルから xset q を再実行し、DPMS is Disabled と prefer blanking: no の 2 つが揃っていれば「X 側」も無効化済み（Screen Saver: timeout: 300 が残っていても問題なし。理由は注意点を参照）。

4. ログインのたびに xset を自動実行させる（~/.xsessionrc）

手順 2＋手順 3 だけで止まる環境がほとんどだが、更新で電源管理が再有効化されたり、X 組み込みスクリーンセーバ（xset s）が独立に動いたりすることもある。保険として、X セッションが開始されるたびに xset を流す 仕組みを入れておく。

※ ここで設定するのは X 組み込みの DPMS とスクリーンセーバへの保険 であって、xfce4-screensaver には効かない（あれは手順 3 で止める仕組み）。

Debian 系（Kali を含む）の GUI ログイン経路で確実に読まれるのは、ホームディレクトリの ~/.xsessionrc である。~/.bashrc や ~/.profile ではなく、これを作る。

テキストエディタで新規作成する：

kali@kali:~$ nano ~/.xsessionrc

ファイルの中身として、次の 3 行を入力する：

# X セッション開始時に画面消灯を抑止する
xset -dpms       # DPMS（モニタ電源管理）を無効化
xset s off       # X 組み込みスクリーンセーバのタイマー停止
xset s noblank   # ブランク描画も停止

保存して閉じる（nano の場合）：

  • Ctrl+O を押し、続けて Enter（ファイル名そのままで保存）
  • Ctrl+X でエディタを終了

実行ビット（chmod +x）は不要。X セッション起動スクリプト（/etc/X11/Xsession）が ~/.xsessionrc を source で読み込んでくれる仕組みだから、シェルスクリプトとして実行可能である必要はない。

反映には一度ログアウトして再ログインする：

  • 画面右上のユーザー名／電源アイコン →「ログアウト」→「ログアウト」を選ぶ
  • lightdm のログイン画面が表示されたら、再度ログインする

再ログイン後、ターミナルを開いて確認：

kali@kali:~$ xset q

出力中に次の表記があれば成功：

  • DPMS is Disabled
  • Screen Saver: timeout: 0 かつ prefer blanking: no

5. それでも黒くなる場合：light-locker / xscreensaver を確認する

現行 Kali Xfce では通常 xfce4-screensaver が画面ロッカー役を担っており、light-locker や xscreensaver は既定では入っていない。ただし古い手順を踏襲して入れた環境、別ディストリから引っ越した環境では、これらが独立に「真っ黒＆ロック」をかけてくることがある。プロセスが動いているか確認する：

kali@kali:~$ pgrep -l light-locker
kali@kali:~$ pgrep -l xscreensaver

何も表示されなければ、両方とも動いていない（＝この手順は無視してよい）。番号付きで名前が出れば、そのプロセスが動いている。

light-locker が動いている場合：いま動いているものを停止：

kali@kali:~$ pkill light-locker

自動起動から外したい場合は「アプリケーション」メニュー →「設定」→「セッションと起動」→「自動開始アプリケーション」タブを開き、light-locker の行のチェックを外す。

xscreensaver が動いている場合：いま動いているものを止める：

kali@kali:~$ xscreensaver-command -exit

次回ログイン時にも起動させたくない場合は、同じく「セッションと起動」の「自動開始アプリケーション」タブから xscreensaver のチェックを外す。

注意点

  • 「xset q は DPMS is Disabled なのにまだ黒くなる」ときは、ほぼ xfce4-screensaver が原因。手順 3 を実施する。それでも黒くなる場合は pgrep -l screensaver や pgrep -l locker でほかに同種のプロセスが動いていないか確認する。
  • xset q の Screen Saver: timeout: 300 が残ったままになるのは正常。これは X 組み込みスクリーンセーバの「タイマー値が 300 秒」という記録に過ぎず、同じ行に prefer blanking: no がある限り、満了しても画面はブランク（黒）にされない。xfce4-screensaver を手順 3 で OFF にしていれば 5 分以上経っても黒くならない。タイマー値の方も 0 にしたい場合は、手順 4 の ~/.xsessionrc に xset s off を入れる（見た目の好みの問題で、挙動上は不要）。
  • xfce4-power-manager と xfce4-screensaver は別パッケージ・別プロセス。「電源管理」設定アプリと「スクリーンセーバの管理」設定アプリがそれぞれ独立した GUI を持つので、どちらか片方だけ無効化しても黒画面は止まらない。両方 OFF にする必要がある。
  • なぜ .bashrc / .profile / /etc/rc.local に書いても効かないのか、3 行まとめ：.bashrc は GUI ログイン経路では実行されない／.profile は lightdm から Xfce を起動する経路では読まれない／/etc/rc.local は X 起動前に走るので xset がそもそも繋がる先が無い。GUI セッション開始時に確実に読まれる場所は ~/.xsessionrc。
  • GNOME / KDE では別のデーモンになる。Kali を Xfce 以外の派生版で使っている場合、画面を消してくる主犯は xfce4-* ではなく gnome-settings-daemon（GNOME）／powerdevil（KDE）と各環境のスクリーンロッカー（gnome-screensaver ／ ksmserver 配下のスクリーンロッカー）になる。GUI 設定の場所は各デスクトップ環境の「電源管理」「省電力」「画面ロック」設定にあるので、そこで「画面の電源を切る」「ブランクスクリーン」「自動ロック」を 無効 ／しない に変更する。~/.xsessionrc の方は同じファイルでそのまま使える。
  • VirtualBox 上で動かしている場合、ホスト OS 側の電源設定も影響する。Kali（ゲスト）の DPMS とスクリーンセーバを完全に止めても、Windows（ホスト）の「設定」→「電源とバッテリー」→「画面とスリープ」で「画面をオフにする」が短時間に設定されていると、ホストの画面が消えてゲストも見えなくなる。両方の設定を見ること。
  • Wayland セッションでは xset 自体が効かない。Kali Xfce のデフォルトは X11 なので通常は関係ないが、念のため確認したい場合はターミナルで echo $XDG_SESSION_TYPE を実行し、出力が x11 なら xset 系の手順 4 がそのまま効く。wayland と出る場合は手順 4 が無効で、GUI 設定（手順 2＋手順 3）のみで止める必要がある。
  • 学習用 VM 向けの設定であり、業務環境ではそのままコピーしないこと。画面の自動ブランク／自動ロックは席を離れた間の覗き見対策として重要な機能。共用 PC や業務利用では「ディスプレイ電源管理（手順 2）とスクリーンセーバの自動発動（手順 3）は OFF にして黒画面は抑止するが、xfce4-screensaver の『ロック画面』タブの 自動ロックは残す」運用も検討する。
